#修改说明
本项目服务于minisat2.2.0版本的修改，目的是为了适配gcc9.4.0版本下正常编译

##修改原因
  C++11 兼容性: 新版本 GCC 要求字符串字面量与宏之间必须有空格
  默认参数声明: 友元函数声明中的默认参数需要在定义中明确
  函数调用一致性: 修复因默认参数声明变化导致的调用错误

##修改内容
1. 修复字符串字面量与宏的连接问题
文件: utils/Options.h
修改: 在 PRIi64 宏前添加空格，使其符合 C++11 标准
// 原内容:
fprintf(stderr, "%4"PRIi64, range.begin);
fprintf(stderr, "%4"PRIi64, range.end);
fprintf(stderr, "] (default: %"PRIi64")\n", value);
// 修改为:
fprintf(stderr, "%4" PRIi64, range.begin);
fprintf(stderr, "%4" PRIi64, range.end);
fprintf(stderr, "] (default: %" PRIi64 ")\n", value);

2. 修复友元函数默认参数声明问题
文件: core/SolverTypes.h
修改: 恢复 mkLit 函数的默认参数声明
//原内容:
friend Lit mkLit(Var var, bool sign);
// 修改为:
friend Lit mkLit(Var var, bool sign = false);

3. 修复 mkLit 函数调用缺少参数的问题
文件: core/Dimacs.h
修改: 为 mkLit 调用添加第二个参数
// 原内容:
lits.push( (parsed_lit > 0) ? mkLit(var) : ~mkLit(var) );
// 修改为:
lits.push( (parsed_lit > 0) ? mkLit(var, false) : ~mkLit(var, false) );
